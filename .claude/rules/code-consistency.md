## Code Consistency Rule

기존 패턴과 일관성이 깨지는 코드를 생성하기 전에 반드시 플래그하고 논의한다.

체크 대상: import 스타일 변경 · 에러 처리 방식 변경 · 네이밍 패턴 변경 · Response 형태 변경 · Guard/Decorator 위치 변경

플래그 형식: "⚠️ 일관성 체크 — 기존: X / 변경: Y / 이유: Z — 진행할까요?"

예외: 기존 코드와 겹치지 않는 새 기능 추가 · 같은 세션에서 이미 논의한 경우

## Rule of Three

같은 로직이 세 번째 등장하려는 순간 → 생성 멈추고 플래그.

플래그 형식: "⚠️ Rule of Three — [패턴] 세 번째 중복입니다. 추출할까요?"
